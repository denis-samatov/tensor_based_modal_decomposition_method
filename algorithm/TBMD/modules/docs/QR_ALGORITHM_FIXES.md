# Исправления алгоритма Tensor QR Decomposition

## Обнаруженные проблемы

### 1. **Неправильное применение преобразований Хаусхолдера**

**Проблема:** В исходном коде преобразования Хаусхолдера применялись неправильно к тензорам R и Q.

**Оригинальный код:**
```python
# НЕПРАВИЛЬНО
Ru = sub_R @ u  # (spatial_size,) vector
sub_R -= 2 * Ru.unsqueeze(1) * u.unsqueeze(0)
```

**Исправленный код:**
```python
# ПРАВИЛЬНО - согласно Algorithm 2
uT_tubes = flat_R @ u  # (spatial_size,) - dot product of u with each tube
flat_R -= 2 * uT_tubes.unsqueeze(1) * u.unsqueeze(0)
```

**Объяснение:** 
- Согласно Algorithm 2: `R_{x,y,d:k} ← R_{x,y,d:k} - 2u^T R_{x,y,d:k}`
- Для каждой пространственной позиции (x,y), трубка (tube) `t = R[x,y,d:k]` преобразуется как: `t_new = t - 2 * (u^T @ t) * u`
- Исходный код применял преобразование в неправильном направлении

### 2. **Слишком строгие пороги численной точности**

**Проблема:** Пороги были установлены слишком строго для типа данных float32.

**Изменения в конфигурации:**
```python
# БЫЛО (слишком строго)
ORTHOGONALITY_TOLERANCE: float = 1e-8

# СТАЛО (реалистично для float32)
ORTHOGONALITY_TOLERANCE: float = 5e-6
```

**Обоснование:**
- float32 имеет машинную точность ~1e-7
- При 200 итерациях накопительные ошибки округления достигают ~1e-6
- Требование точности 1e-8 нереалистично для float32

### 3. **Некорректная валидация N > k**

**Проблема:** Алгоритм падал с ошибкой при N > k, хотя это допустимая ситуация.

**Исправление:**
```python
# БЫЛО
if N > k:
    raise ValueError(f"N ({N}) cannot exceed tube dimension k ({k})")

# СТАЛО
if N > k:
    print(f"Warning: N ({N}) exceeds tube dimension k ({k}). "
          f"Only {k} sensors can be effectively placed.")
```

**Обоснование:**
- Математически можно запросить больше сенсоров, чем временных измерений
- Алгоритм просто завершится досрочно при достижении k итераций
- Это предупреждение, а не критическая ошибка

## Математическое обоснование исправлений

### Формула Хаусхолдера из Algorithm 2

Согласно статье, преобразование Хаусхолдера определяется как:

1. **Вычисление нормы:** `σ ← ||t||₂`

2. **Вычисление вектора:** `u ← (t + sign(t₁)σe^d) / √(2σ(σ + |t₁|))`

3. **Применение к R:** `R_{x,y,d:k} ← R_{x,y,d:k} - 2u^T R_{x,y,d:k}`

4. **Применение к Q:** `Q_{:,d:k} ← Q_{:,d:k} - 2Q_{:,d:k}uu^T`

### Ключевые моменты

- **Направление применения:** Преобразование применяется к каждой трубке индивидуально
- **Векторизация:** Для эффективности все трубки обрабатываются одновременно
- **Численная стабильность:** Используется формула, избегающая катастрофической потери точности

from TBMD.modules.TensorBasedTubeFiberPivotQRFactorization import (

    ORTHOGONALITY_TOLERANCE=1e-5,  # Реалистично для float32

    tensor=your_tensor,
