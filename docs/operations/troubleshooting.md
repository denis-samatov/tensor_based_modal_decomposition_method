# Troubleshooting

## Purpose
Provides guidance on resolving common issues when running the TBMD pipeline.

## Audience
Engineers and Researchers.

## Summary
Most issues arise from tensor shape mismatches, memory exhaustion, or poor mathematical convergence.

## Details

### 1. Shape Mismatch During Testing
- **Error**: `ValueError: Expected 4D tensor but got 3D`
- **Cause**: Input tensors to the pipeline must match the configuration (e.g., `[features, x, y, time]`).
- **Solution**: Ensure your data loader reshapes the numerical arrays correctly before calling `.fit()`.

### 2. OOM (Out of Memory) during Decomposition
- **Error**: Process killed by OS or CUDA out of memory.
- **Cause**: Unfolding a massive spatiotemporal tensor into a matrix for SVD exceeds available RAM.
- **Solution**: Use `BatchModalProcessor` to compute modes incrementally, or reduce the spatial grid resolution.

### 3. High Reconstruction Error
- **Error**: Reconstructed tensors drastically differ from ground truth.
- **Cause**: Selected ranks are too low, or the physical sensors are placed in non-informative regions.
- **Solution**: Increase tensor ranks in the configuration, or visualize the sensor locations generated by `QRBasedSensorPlacement` to ensure they capture key dynamic regions.
